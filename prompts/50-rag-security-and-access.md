# 50-rag-security-and-access

<context>
Классифицировать данные, описать маскирование и правила доступа к датасету для разных ролей.
</context>

<inputs>
- `target/rag_dataset_spec/02_info_needs.md`
- `target/rag_dataset_spec/03_sources_and_scope.md`
- `target/rag_dataset_spec/04_schema_and_metadata.md`
</inputs>

<outputs>
- `target/rag_dataset_spec/06_security_and_access.md` — классы данных, маскирование, правила доступа/исключения по ролям.
- PDR: `pdrs/YYYY-MM-DDTHH-MM-SS_50-rag-security-and-access.md`
</outputs>

<steps>
1) Определить классы данных (public/internal/confidential/restricted) применительно к источникам и полям.
2) Указать, какие классы входят в датасет для операторов/внутренних пользователей/клиентов; что исключено.
3) Описать требования к маскированию/редакции/фильтрации документов и полей.
4) Зафиксировать правила контроля доступа на уровне документов/метаданных, включая аудит и наследование прав из источников.
5) Подсветить риски и принятые решения (trade-offs), связанные с доступностью vs безопасностью.
</steps>

<constraints>
- Не предлагать конкретные инструменты DLP/IRM; только требования и ограничения.
- Любые допущения по рискам/регуляторике должны быть явно названы.
</constraints>

<audit_log>
- В PDR отметить, какие классы данных исключены, какие риски приняты/сняты, какие артефакты обновлены.
</audit_log>
