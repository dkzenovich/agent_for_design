# input/

Сырые входные данные для RAG-датасета (без редактуры):
- `01_business_context.md` — кто пользователь, зачем RAG, примеры задач, требования по свежести/рискам.
- `02_example_questions.md` — реальные/ожидаемые вопросы и диалоги, частота, важность.
- `03_sources_inventory.md` — список систем/хранилищ, типы документов/таблиц, правовые/ИБ-ограничения.

Можно добавлять любые вспомогательные выгрузки и заметки (текст/CSV/скриншоты) — агент должен читать их как есть.
